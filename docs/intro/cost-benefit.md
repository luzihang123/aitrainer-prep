# 备考性价比分析

> 一句话结论：**Python 编程是绝对重点（70+ 分），Excel 是性价比之王（15 分易拿满），理论部分有套路（30 分能背就背），第四部分（5 分）最后再看**。

## 性价比四象限

| 象限 | 内容 | 性价比 | 投入策略 |
|------|------|-------|---------|
| 🟢 高分 + 易拿 | Excel（3.1.x，15 分） | ★★★★★ | 必须吃透 |
| 🟢 高分 + 易拿 | Python 数据预处理（~40 分） | ★★★★☆ | 重点投入 |
| 🟡 高分 + 中等难度 | Python 建模（2.2.x，20 分） | ★★★★☆ | 套路化、可背 |
| 🟡 高分 + 中等难度 | Python 图像识别（3.2.x，20 分） | ★★★☆☆ | 流程化、API 多 |
| 🟠 中分 + 难 | 理论（30 分） | ★★★☆☆ | 模板化、可背 |
| 🔴 低分 + 杂 | 第四部分（4.x，5 分） | ★★☆☆☆ | 最后再看 |

## 为什么 Excel 是性价比之王

Excel 题（3.1.x）有几个显著特点：

- **操作有限**：考的就那十几个常用功能（VLOOKUP、SUMIF、数据透视表、条件格式等）
- **答案唯一**：操作正确就给分，不像理论题有主观成分
- **不依赖代码记忆**：不需要默写 API
- **占 15 分**：相当于一道大题的分值

**结论**：花 2-3 天把 Excel 这部分吃透，能稳拿 15 分，相当于及格分的 25%。

## 为什么 Python 编程是重点

把 Python 相关的部分加起来：

- 第一部分如果命中 1.1.x（Python 题）：25 分
- 第二部分 2.1.x + 2.2.x：35 分
- 第三部分 3.2.x（图像识别）：20 分
- **合计最多 80 分，最少 55 分**

**所以 Python 是这个考试的命脉。** 但这不意味着要把 Python 学得多深——

::: tip 关键认知
**题目不提供代码框架，你只需要背诵 import 语句和核心 API 调用。**

这意味着备考重点不是"会写程序"，而是"能在白纸上默写出 10 行常用代码"。完全靠记忆，不靠理解。
:::

## 为什么理论部分有套路

理论部分（1.2.x + 4.x，最多 30 分）看着主观，其实**有固定模板**。

例如 1.2.x 的"问题对策"题目，本质就是把已知问题对应到已知对策的填空题：

| 问题类型 | 对策方向 |
|---------|---------|
| 准确性不高 | 优化数据质量 / 优化模型算法 |
| 响应慢 / 延迟高 | 优化模型算法 / 优化系统性能 |
| 缺乏个性化 / 服务单一 | 优化用户体验 |

**记忆口诀**：「准确不高靠数据 + 算法，响应慢靠算法 + 系统，单一靠体验」。

详见 [理论模板速查](/cram/theory-templates)。

## 推荐的精力分配

如果你有 30 天准备：

```
第 1 天     ：通读本站、定计划、背 import 语句速查
第 2-4 天   ：吃透 Excel（拿稳 15 分）
第 5-12 天  ：Python 数据预处理（拿稳 ~40 分）
第 13-17 天 ：Python 建模（拿稳 20 分）
第 18-22 天 ：Python 图像识别（拿稳 20 分）
第 23-26 天 ：理论模板背诵（争取 ~25 分）
第 27-29 天 ：刷题、查漏、听题模式
第 30 天    ：考前 checklist
```

按这个节奏走完，**理论及格分 60、目标分 80+**。
