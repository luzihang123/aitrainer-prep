# Prompt: 知识图谱关联（related）

## 任务

把这道题接入项目的**知识网络**——找出它和其他题目的关系。这是项目最长期的资产之一。

## 输出四个字段

```yaml
related:
  same_concept: [<同考点的题目 id 列表>]
  contrast: [<易混概念的题目 id 列表>]
  prerequisite: [<前置知识的题目 id 列表>]
  next_step: [<进阶题目的 id 列表>]
```

## 字段含义

### same_concept（同考点）

考的是**同一个知识点**的其他题目。如果一个考生这道题不会，看 `same_concept` 里的题就是换个角度刷同一个点。

例：本题考"无监督学习定义"，`same_concept` 里应该是其他考无监督学习定义或核心特征的题。

### contrast（易混概念）

考的是和本题**容易混淆**的相邻概念。让考生通过"对比"来辨析。

例：本题答案是"无监督学习"，`contrast` 里可以放"半监督学习"或"监督学习"的相关题，方便考生横向对照。

### prerequisite（前置知识）

理解本题前应该先懂的题。如果某考生看本题完全懵，他可以先去做 `prerequisite` 里的题。

例：本题考"无监督学习"，前置可能是"什么是机器学习"或"什么是标签"的基础题。

### next_step（进阶）

理解本题后**自然该看的下一题**。建立从基础到深入的学习路径。

例：本题考"无监督学习定义"，进阶可以是"无监督学习常见算法"或"K-Means 实现"的题。

## 数量要求

- 每个字段 0-5 个 id
- **空字段也要保留**：用 `[]` 表示没有关联，不要省略键
- 优先填高质量关联，**宁缺毋滥**

## 输入

你会收到：

1. 当前题目的 frontmatter
2. **题库索引摘要**：所有已入库题目的 `id` + `tags` + `question` 第一句

你必须**只从题库索引里挑**——不要凭空编 id。

## 不确定时

如果题库太小（< 20 题）找不到合适的关联，全部填 `[]`。**不要硬编 id**。

## 输出格式

只输出 `related:` 这一段 YAML，不要前后说明、不要代码块包裹：

```
related:
  same_concept: [3.1.015, 3.1.023]
  contrast: [3.1.018]
  prerequisite: [3.1.005]
  next_step: [3.2.002]
```
