# Prompt: 排除法解析（elimination）

## 任务

为每个**错误选项**生成排除法解析。**只讲错的，不讲对的**——正确选项交给 explanation 处理。

## 排除法分类（type 取值）

每个错误选项必须标注一个排除类型：

| 类型 | 含义 | 例子 |
|------|------|------|
| **事实错误** | 选项陈述本身违背事实 | "Python 是一种编译型语言"（错，Python 是解释型） |
| **概念混淆** | 选项是另一个相似概念的定义 | 把"监督学习"的定义贴在"无监督学习"选项上 |
| **不完整** | 选项只对了一半或缺少关键条件 | "提高准确性的方法是优化模型"（漏了优化数据） |
| **范围错** | 选项超出或不到题目所问范围 | 题问"机器学习"，选项说"深度学习的某种"（范围太窄） |
| **偷换前提** | 选项改变了题干的前提条件 | 题干说"无标签数据"，选项隐含"有少量标签" |
| **绝对化** | 选项使用"所有""一定""绝不"等绝对词 | "使用神经网络一定能提高准确率" |

## 输出格式

YAML 数组，每个错误选项一项：

```yaml
elimination:
  - option: A
    type: 概念混淆
    reason: <为什么用这个类型 + 具体怎么错——一句话讲清>
  - option: C
    type: 范围错
    reason: ...
```

## 风格要求

- **每条理由 ≤ 30 字**，简洁、可朗读
- **不要复述选项内容**——只讲为什么错
- **用第二人称**，像在指点考生
- 不用"值得注意的是"等套话

## 好坏对比

题目：下列哪种学习方式不需要标签数据？答案：B 无监督学习

- ✅ 好：
  ```yaml
  elimination:
    - option: A
      type: 概念混淆
      reason: 监督学习需要标签，与题干"不需要标签"直接矛盾
    - option: C
      type: 范围错
      reason: 强化学习有奖励信号但不属于"标签"概念
    - option: D
      type: 不完整
      reason: 半监督学习仍部分依赖标签，不符合"不需要"
  ```

- ❌ 差：
  ```yaml
  elimination:
    - option: A
      type: 错误
      reason: 监督学习是一种使用带有标签的训练数据来训练模型，使模型能够预测新数据的标签的机器学习方法，因此需要标签数据，与题目要求不符。
  ```
  （问题：type 不规范、reason 太长且复述定义）

## 不能做的事

- ❌ 不要分析正确选项（那是 explanation 的活）
- ❌ 不要让 reason 超过 30 字
- ❌ 不要发明排除法分类——必须从上表 6 种里选

## 输出

只输出 `elimination:` 这一段 YAML，不要任何前后说明、不要包裹代码块。

## 输入

题目的 frontmatter（必须含完整的 `options` 和 `answer`）。
