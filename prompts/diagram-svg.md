# Prompt: SVG 静态图解

## 任务

为给定题目生成一张 SVG 图解，帮助读者**视觉化理解**核心概念。

## 适用场景

- 概念对比（如监督 vs 无监督学习）
- 结构示意（如神经网络层级）
- 关系图（如各种学习方式的分类）
- 静态状态（不涉及流程时间线）

涉及流程 / 决策树用 [diagram-mermaid.md](./diagram-mermaid.md) 而不是 SVG。

## 技术要求

- viewBox 统一使用 `0 0 600 400`
- 不依赖外部字体（用 system-ui 或 sans-serif）
- 颜色用 CSS 变量或具名色（不要绑死 hex）
- 文字必须可读（≥ 12px）
- 图形必须语义化（用 `<title>` 和 `<desc>`）
- 不要复杂滤镜、阴影、渐变（保持简洁、可在深色模式下也清晰）

## 风格要求

- **简洁**：能用 5 个元素说清楚就不用 10 个
- **平面化**：不要立体效果、不要装饰
- **中文标签**：直接用中文，不需要英文对照
- **配色克制**：最多 3 种颜色

## 输出格式

只输出 SVG 代码本身，不要任何前后说明、不要包裹在代码块里：

```svg
<svg viewBox="0 0 600 400" xmlns="http://www.w3.org/2000/svg">
  <title>图解标题</title>
  <desc>图解的描述（用于无障碍）</desc>
  <!-- 图形元素 -->
</svg>
```

## 不能做的事

- ❌ 不要生成涉及版权角色、品牌 logo 的图
- ❌ 不要堆砌装饰元素
- ❌ 不要写 `<script>` 标签
- ❌ 不要写复杂的 `<style>` 块——直接用属性

## 输入

题目的 frontmatter，可能还有已生成的 `explanation`。
