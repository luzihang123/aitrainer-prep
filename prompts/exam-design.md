# Prompt: 出题人视角（exam-design）

## 任务

进入**出题人的脑子**，分析这道题为什么被设计出来。这是本站和其他备考资料的核心差异化——不只告诉考生答案，而是教他们识别出题人的套路。

## 输出三个字段

```yaml
exam_design:
  core_concept: <核心考点是什么——一句话>
  difficulty_role: <送分题 / 区分题 / 压轴题>
  trap_design: <出题人怎么挖坑——一句话>
```

## 字段含义详解

### core_concept（核心考点）

题目想确认考生掌握了哪个具体知识点。**必须是具体的、可教学的、可背诵的一个点**，不要写"机器学习基础"这种泛泛的话。

- ✅ 好：监督学习与无监督学习的本质区别（是否需要标签）
- ❌ 差：机器学习相关知识

### difficulty_role（难度定位）

判断这道题在试卷里扮演什么角色：

- **送分题**：稍微学过的人就该会，不会就 hold 不住基础
- **区分题**：能区分"懂"和"似懂非懂"的考生
- **压轴题**：有陷阱、有细节、需要深入理解才能选对

判断依据：
- 概念是否一线（核心 vs 边缘）
- 选项是否有高欺骗性的干扰项
- 是否需要多步推理

### trap_design（干扰策略）

出题人**在错误选项里挖了什么坑**？

- ✅ 好：用"半监督"作为干扰，考生若只记"无标签"会犹豫
- ✅ 好：列出 3 个看起来都合理的对策，但只有 1 个对应该问题类型
- ❌ 差：选项之间差别很明显（这是"差"——意味着出题人没挖坑，那这题就是送分题）

## 风格要求

- **每个字段一句话**，最多两句
- **直接、不绕弯**
- 不用"值得注意的是"等套话

## 输出格式

只输出三个 YAML 字段，不要包裹在代码块里：

```
exam_design:
  core_concept: ...
  difficulty_role: ...
  trap_design: ...
```

## 输入

题目的 frontmatter。
